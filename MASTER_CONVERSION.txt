SECTION 3 — Phase 1: Setup & Scaffolding



Step 1.1 — Create the Next.js app inside the subfolder
Open a terminal in your repo root. Run these commands exactly:
cd YOUR-REPO-ROOT
npx create-next-app@latest nextjs-app \
  --typescript \
  --tailwind=false \
  --eslint \
  --app \
  --src-dir=false \
  --import-alias="@/*"

When prompted, answer:
Would you like to use Tailwind CSS? → No
Would you like to use src/ directory? → No
Would you like to customize the default import alias? → No

Step 1.2 — Copy Assets to public/
Copy all images, fonts, icons from the original repo into nextjs-app/public/. Keep the same subfolder structure.
cp -r ../assets/* ./nextjs-app/public/
If your original site has images inline in HTML (e.g. src="images/hero.jpg"), they will map to /images/hero.jpg in Next.js public folder.

Step 1.3 — Configure next.config.js
Open nextjs-app/next.config.js and replace its contents with:
/** @type {import('next').NextConfig} */
const nextConfig = {
  output: 'export',  // static export for Vercel
  trailingSlash: true,
  images: { unoptimized: true },
};
module.exports = nextConfig;

Step 1.4 — Test the scaffold runs
cd nextjs-app
npm run dev
Open http://localhost:3000 — you should see the default Next.js welcome page. That is fine at this stage.

SECTION 5 — Phase 3: Convert CSS



Step 3.1 — Create globals.css
Copy your original style.css content into nextjs-app/app/globals.css. Rules:
Keep ALL selectors, ALL rules exactly as they are
Do NOT add Tailwind, do NOT refactor, do NOT rename classes
Only change: url() paths for fonts/images must be updated to /fonts/ or /images/
CSS variables (:root { }) stay in globals.css
@import statements for Google Fonts stay as-is

Step 3.2 — Import in layout.tsx
At the top of nextjs-app/app/layout.tsx, add:
import './globals.css';

Step 3.3 — Multiple CSS Files
If original site has multiple CSS files (style.css, animations.css, etc.), merge them all into globals.css in the same order they were linked in the original HTML <head>.

Step 3.4 — Component-Specific Styles
Only create .module.css files if a component needs styles that would conflict globally. For a simple site, everything in globals.css is fine.

Step 5.1 — HTML to JSX Conversion Rules

Step 5.2 — Page File Template

Step 5.3 — Images
Simple rule: use a plain <img> tag first. It will work. Only switch to next/image if you want automatic WebP optimization. For exact visual parity, plain <img> is safer.

Step 5.4 — Links
Replace ALL <a href="page.html"> with Next.js <Link href="/page">. Import at top:

SECTION 8 — Phase 6: Convert JavaScript



Step 6.1 — Identify JS Type
Read the original script.js and classify each piece:
DOM manipulation on page load (e.g. adding classes) → useEffect
Event listeners (click, scroll, resize) → useEffect + addEventListener
Animations / scroll effects → useEffect
Form validation → useState + inline event handlers
Data fetching → async useEffect or React Server Component
Simple utility functions (no DOM) → keep as plain TS functions

Step 6.2 — useEffect Pattern
When original JS runs on DOMContentLoaded:
'use client';  // ← required at top of file if using hooks
import { useEffect } from 'react';

export default function MyPage() {
  useEffect(() => {
    // PASTE original JS code here
    // Replace document.querySelector with ref if needed
  }, []);  // empty deps = runs once on mount
}

Step 6.3 — Mobile Menu / Navbar Toggle
This is the most common JS pattern. Convert like this:
'use client';
import { useState } from 'react';

export default function Navbar() {
  const [menuOpen, setMenuOpen] = useState(false);
  return (
    <nav>
      <button onClick={() => setMenuOpen(!menuOpen)}>Menu</button>
      <ul className={menuOpen ? 'open' : ''}>...
    </nav>
  );
}

Step 6.4 — 'use client' Rule
Add 'use client' at top of ANY file that uses useState, useEffect, onClick, or window/document
Server Components (no interactivity) do NOT need 'use client'
layout.tsx does NOT need 'use client' unless it has interactive elements

SECTION 10 — Phase 8: Deploy to Vercel



Step 8.1 — Push to GitHub
git add .
git commit -m 'Phase 8: Ready for Vercel deployment'
git push origin main

Step 8.2 — Vercel Project Settings (CRITICAL)
When importing your repo on Vercel, you MUST configure the subfolder:
Go to vercel.com → Add New Project → Import your GitHub repo
In 'Configure Project' settings:
Root Directory: set to   nextjs-app   (click Edit → type nextjs-app)
Framework Preset: Next.js (auto-detected)
Build Command: npm run build  (default — no change needed)
Output Directory: leave blank (Next.js handles it)
Click Deploy

Step 8.3 — vercel.json (Alternative Method)
If you prefer, add a vercel.json in your REPO ROOT (not inside nextjs-app):
{
  "buildCommand": "cd nextjs-app && npm run build",
  "outputDirectory": "nextjs-app/.next",
  "installCommand": "cd nextjs-app && npm install",
  "framework": "nextjs"
}

Step 8.4 — Verify Deployment
Vercel dashboard shows green 'Ready' status
Visit the .vercel.app URL
All pages load correctly
All images display
All links work
Mobile view matches original

Step 8.5 — Custom Domain (Optional)
Vercel Dashboard → Your Project → Settings → Domains
Add your domain and follow DNS configuration instructions
SSL is automatic
